# Dev Clipboard MVP Checklist

Last updated: 2026-07-14

この文書を Phase 1 MVP の進捗と完成条件の基準にする。`../../tech-plan.md` と `../../handoff-jp.md` の範囲を、現在の実装に照らして整理している。

## MVP Definition

MVPの中心価値は、コピーした開発情報をローカルに保存し、`Description`、`When to use`、`Before`、Riskを確認・検索してから、クリップボードへ安全に戻せること。

MVP達成は「画面が存在する」ではなく、以下を満たした状態とする。

- macOSで日常的に起動し、ショートカットから安定して呼び出せる。
- 取得、保存、検索、編集、コピー、削除、Undoの主要フローでデータ不整合が起きない。
- 設定画面に、動作しない通常コントロールを残さない。未実装項目は明確に `Future` とする。
- ライト・ダーク、Compact・Normal・Largeで主要UIが崩れない。
- 危険コマンドの判定限界を明示し、Risk表示を過信させない。
- 配布用ビルドを作成し、クリーンな環境で基本フローを確認できる。

Status:

- `[x]` 実装済み。MVP向けの最終確認は別途必要。
- `[~]` 一部実装。完成条件に不足がある。
- `[ ]` 未実装または未検証。
- `[-]` MVP対象外。

## MVP Blockers

以下はMVP達成前に完了が必要。

### P0: データと安全性

- [x] `DEMO_HEAVY_CLIPS` の自動投入を停止。開発版Settingsから明示的に追加・デモだけ削除でき、`[Demo]`タイトルと`is_demo`フラグで実履歴と区別する。
- [ ] 履歴全削除を、対象範囲の表示・確認ダイアログ・SQLite/FTS同時削除・失敗表示付きで実装する。
- [ ] コピー元アプリを実取得するか、Ignored appsをMVPから外すか決定する。設定に実装済みのような表示を残さない。
- [ ] パスワード、秘密鍵、`.env`など機密情報の保存方針を決め、最低限の警告または除外策を用意する。
- [ ] Risk判定ルールの自動テストを追加する。少なくとも `rm -rf`、`sudo`、`git reset --hard`、Docker削除系と安全な類似文字列を含める。
- [ ] DB操作を可能な範囲でトランザクション化し、`clips` と `clip_search` の片方だけが更新される状態を防ぐ。

### P0: 日常利用の信頼性

- [ ] 50件表示上限より古い履歴へ到達できるよう、ページングまたは追加読み込みを実装する。
- [ ] クリップボード監視、SQLite、検索、コピー失敗を、消えるステータス文字列だけでなくユーザーが認識できるエラー状態として表示する。
- [ ] パネルがフォーカスアウトで退避するとき、メモ編集中・設定中・検索Popover操作中の状態を保持して復帰できることを確認する。
- [ ] アプリ再起動後も Theme、Card size、必要な検索・表示設定が期待どおり復元されることを確認する。ThemeとCard sizeは保存実装済み。
- [ ] 設定画面の通常表示コントロールを実機能へ接続するか、`Future`表示へ変更する。

### P0: リリース品質

- [ ] `App.tsx` の分類、DB、表示責務を分割し、テスト可能な純粋関数へ切り出す。
- [ ] コアフローの自動テストを追加する。最低限、分類、Risk、検索用テキスト、色変換、タイトル生成を対象にする。
- [ ] 手動QAチェックリストを作り、Dark / Light × Compact / Normal / Largeで実施する。
- [ ] 長い英数字、日本語、複数行、URL、色、コマンドで横overflowと重なりがないことを確認する。
- [ ] Tauriの配布用ビルドを作成し、アプリ名、identifier、アイコン、バージョン、権限表示をMVP用に確定する。
- [ ] macOS署名・Notarizationを行うか、テスト配布方法を明文化する。
- [ ] `csp: null` と `macOSPrivateApi: true` の必要性を再確認し、配布時のセキュリティ設定を確定する。

## P1: MVP Polish

P0完了後、MVP候補版を他者に渡す前に対応する。

- [~] Settings: Themeのみ実動作。Card sizeは実動作と永続化に接続済み。他のモック項目を整理する。
- [~] Source app表示: 現在は内容由来の推定・モック。実際の前面アプリ取得とアイコン表示は未実装。
- [~] Compact / Normal / Large: 切替とCard size保存は実装済み。全カード種・全テーマの実機QAが未完了。
- [~] メディアカード: Image / Video / Audio / SVG / Illustrator / PDF / File / URL / Colorの表示ルールはUI検証中心。実クリップボードデータ取得はtext-firstの範囲外。
- [~] Color: HEX / RGB / HSL / RGBAの表示変換がある。原文保持、alpha、無効値、Copy形式の回帰確認が必要。
- [ ] リッチテキストをプレーンテキストとして取り込む最小方針を決める。高度な編集はMVP外。
- [ ] HelpにLocal only、Auto Pasteなし、Risk判定の限界、内部コピー除外、保存場所を記載する。
- [ ] VoiceOver名、キーボードフォーカス順、コントラスト、Reduced Motionを確認する。
- [ ] 空状態、初回起動、DBエラー、検索0件、削除後、クリップボード権限失敗の文言を整える。

## MVP Completion Gate

次をすべて満たした時点で「MVP達成」と判断する。

1. P0項目がすべて完了している。
2. `npm run build`、Rust側テスト、Tauri配布ビルドが成功する。
3. 新規DBと既存DBの両方で起動・保存・検索・編集・削除・Undoが成功する。
4. Dark / Light × Compact / Normal / Largeの手動QAに重大な表示崩れがない。
5. 危険コマンドの代表ケースでReview/Riskが表示され、安全なケースを過度にRisk扱いしない。
6. macOS上でショートカット、フォーカス退避、スリープ復帰、再起動後の永続化を確認する。
7. 未実装機能は通常操作できるように見せず、`Future`またはMVP外として明示されている。

## Explicitly Out of MVP

- [-] Auto Paste / 前面アプリへの自動貼り付け
- [-] MCP連携
- [-] クラウド同期
- [-] AIによるメモ自動生成
- [-] セマンティック検索 / ベクトル検索
- [-] 画像クロップや高度なリッチデータ編集
- [-] Plain modeの本格実装
- [-] チーム共有
- [-] ログイン、課金、ライセンス管理
- [-] 完全なCursor / VS Codeテーマ取り込み

## Recommended Order

1. 履歴全削除、機密情報方針、Ignored appsのMVP判断。
2. DB整合性、古い履歴へのアクセス、監視・エラー復帰を強化。
3. Settingsの実機能とFuture表示を整理。
4. 純粋関数の切り出しと自動テストを追加。
5. 全テーマ・全カードサイズの手動QAとUI修正。
6. アプリ識別情報、アイコン、配布ビルド、署名方針を確定。
